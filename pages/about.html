// why this exists

Not a product. A rehearsal.

Most portfolio sites are the deliverable. Here, the site is deliberately simple — the pipeline around it is the actual project.

The problem with "I deployed a static site"

Deploying a folder to a host proves you can click a button. It doesn't show whether you catch a broken link before a user does, whether a leaked API key gets flagged, or whether a bad deploy gets rolled back automatically.

What this project proves instead

Every change to this site goes through validation, an automated build, a test pass, a security scan, a deployment, and a post-deploy verification step — the same shape as a real production release process, just scaled down.

Operational principles applied

Fail fast on bad input, never deploy something untested, treat secrets scanning as non-negotiable, verify the thing you shipped actually works in production, and log every incident that slips through so the pipeline gets stricter over time.
