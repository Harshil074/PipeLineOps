Keeping the pipeline cheap

This project runs entirely on GitHub's free tier, but the habits below are the ones that matter once a pipeline runs hundreds of times a day.

~40% faster

Cache dependencies between runs

Restoring a cached node_modules instead of reinstalling on every run cuts several minutes off each workflow — real money at CI-minute scale.

Fewer runs

Path filters on triggers

Only run the full pipeline when docs/**.md or config files change if they can't break the site; skip straight past stages that can't possibly fail.

Free hosting

Static hosting over a server

GitHub Pages serves this entire site for free with no compute cost per request — appropriate because nothing here needs a backend.

Fail fast

Cheapest checks first

Lint and JSON validation run before the heavier headless-browser test stage, so a typo fails in seconds instead of after a full test suite spins up.

No idle spend

Concurrency limits

New pushes to the same branch cancel the in-flight run for that branch, so pushing three fixes in a row doesn't pay for three full pipeline runs.
