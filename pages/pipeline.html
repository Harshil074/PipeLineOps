What each stage actually does

1. Validate

Lints every HTML, CSS, and JS file and confirms all four JSON data files parse cleanly. Catches typos before they cost a build minute.

2. Build

Writes the current commit SHA, build number, and timestamp into data/version.json, then assembles everything into a single deployable artifact.

3. Test

Spins up a headless browser against the built artifact and checks that each page loads, the nav renders, and the dashboards actually populate from JSON — not just that the HTML is well-formed.

4. Security Scan

Runs a secrets scanner across the diff and audits dependencies for known vulnerabilities. This stage has no override — a real finding blocks the deploy.

5. Deploy

Syncs the build artifact to an AWS S3 bucket configured for static website hosting — no manual FTP, no "it works on my machine."

6. Verify

Curls the live URL, confirms a 200 response, and checks that the deployed version.json matches what was just built. If it doesn't, the run fails loudly instead of quietly serving a stale site.
